HTML 水平线

<hr />标签在 HTML 页面中创建水平线。hr 元素可用于分隔内容。

例子

<p>hr 标签定义水平线：</p>
<hr />
<p>这是段落。</p>
<hr />
<p>这是段落。</p>
<hr />
<p>这是段落。</p>

运行效果

hr 标签定义水平线：

这是段落。

这是段落。

这是段落。

实际上hr解析出来就是没有高度，边框为1px的块元素，注意上边框和下边框的颜色不一致，hr默认的margin为 ”6px 0;“，用户根据需要可修改。

项目中需要注意的地方

表单中label标签

表单的书写格式应该为

<label>姓名 <input /></label>

效果

另一种写法为

<input id="inputId1" name="inputName" type="radio" /> <label for="inputId1">选中</label>
<input id="inputId2" name="inputName" type="radio" /> <label for="inputId2">取消</label>

效果

显然第一种写法更好些，建议采取第一种写法，除非不好控制时，可以采用第二种写法，比如换行

对于输入框（input）限制字符长度的情况，我们必须设置maxlength属性

<input maxlength="5" type="text" />

效果

书写html标签时一定要闭合

尤其是在template中，由于html模板前后可能会有空格存在，在调用$.fn.html方法时在ie7下不能正确解析。

p标签中不要再嵌套其他块标签

W3c的html4.0.1明确规定P标签是不能包含块元素的

The P element represents a paragraph. It cannot contain block-level elements (including P itself).

如果p中含有块元素div，浏览器会解析成

<p></p>
<div></div>
<p></p>

内联元素中不要嵌套块级元素

比如span中不要嵌套p等块级元素

IE6 IE7 不支持 OPTION 和 OPTGROUP 元素的 disabled 属性

在其他浏览器中，如果设置了disabled，则该选项变灰并且不能选择。但在IE6和IE7下，元素仍然是有效的。解决方案：可以利用javascript动态删除或增加option。

IE6 IE7 IE8(Q) 中 BUTTON 元素的 type 属性默认值不是 submit

根据 HTML4.01 规范中的描述，BUTTON 元素创建一个类似 INPUT 元素类型为 button 的按钮控件，但 BUTTON 元素拥有更丰富的呈现效果，如可以在按钮上加入图片等等。 BUTTON 元素同样拥有 type 属性，其取值为 submit、button、reset，分别对应提交按钮、普通按钮、重置按钮。其中 submit 为 type 属性的缺省值。

问题描述：在 IE6 IE7 IE8(Q) 中 BUTTON 元素 type 属性的默认值不是规范规定的 submit 而是 button，则默认没有设定 type 属性时的 BUTTON 元素将创建一个普通按钮，而不是提交按钮。

解决方案：在使用 BUTTON 元素时应明确的为其设定 type 属性，避免因为浏览器默认属性值不同导致按钮类型出现差异。

img标签 替换文本属性（alt）

有些人分不清title和alt属性到底是什么意思，这里简单的说一下。

alt 属性用来为图像定义一串预备的可替换的文本。在浏览器无法载入图像时，浏览器会显示alt属性中定义的值。

title属性是用来定义提示信息的，当鼠标移到img上时，会显示title属性中定义的内容。
